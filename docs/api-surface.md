# Public API Surface

This page is the deliberate, written record of every name re-exported from
`multitrust` (i.e. every entry in `multitrust.__init__.__all__`). Per
`specs/mission.md` principle #7 and `specs/tech-stack.md` constraint #5,
**only what `multitrust.__init__` re-exports is public**. After Phase 5 (1.0
release) this surface freezes under semver; this inventory exists so the
freeze is intentional, not accidental.

## How to read this page

Each entry is grouped by source module (`core`, `manager`, `operators`, …)
and tagged with one of:

| Tag | Meaning |
| --- | --- |
| **App** | First-class application API — end users write code against this. |
| **Integration** | Public, but primarily for integration / extension authors (custom stores, custom rules, custom policies, framework adapters). |
| **Review-before-1.0** | Currently re-exported but with an open question about whether it should remain public. The cell explains why. **Document-only flag** — no code change in this PR. |

Audit scope: the list below was generated by walking
`multitrust.__init__.__all__` (88 names) on the
`spec/phase-0-foundation-hardening` branch.

## Verifying the inventory

```bash
uv run python -c "import multitrust; print('\n'.join(sorted(multitrust.__all__)))"
```

The set of names this command prints must equal the set of rows in this
document. Both directions matter:

- Every public name appears in this inventory with a one-line purpose.
- Nothing in this inventory is missing from `__all__`.

## Open follow-ups

The **Review-before-1.0** rows above are the hand-off list for the next pass.
None require code changes in this PR (per the audit decision: document-only).
The expected resolution path is a follow-up that, for each row, either (a)
keeps it public with a docstring stating the contract or (b) underscore-
prefixes / relocates it and removes from `__all__`.

Tracking-wise, the highest-leverage candidates for relocation are
`opinion_to_beta_parameters` / `beta_to_opinion` (research-level math),
`trust_span` (internal helper), and `RuleEngine` (wrapped by
`RuleBasedCollector`). The lowest-cost defensive moves before freeze are
trimming these.
